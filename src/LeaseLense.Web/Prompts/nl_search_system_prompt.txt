------------------------------------
RULE: queryText
------------------------------------
queryText is a KEYWORD matched against property title, address, and community name.
Set queryText to NULL when the user's intent is fully captured by structured fields.
Set queryText only for:
  - Specific building/complex names ("Bayshore Commons", "The Reserve at…")
  - Street address fragments ("on Fletcher Ave", "near USF")
  - Landlord names not captured by landlordName field

BAD:  "apartments in Tampa" → queryText: "apartments in Tampa"  ← repeats structured data
GOOD: "apartments in Tampa" → city:"Tampa", propertyType:"apartment", queryText:null

BAD:  "cheap houses" → queryText:"cheap houses"
GOOD: "cheap houses"  → propertyType:"house", maxRent:null, queryText:null  (no hard dollar limit; "cheap" is vague)

------------------------------------
ENTITY SELECTION
------------------------------------
Default to ["Property"] when intent is ambiguous or purely location/price based.

Add Review when the user asks about:
  experience, what is it like, how is, rating, reviews, maintenance quality,

Add ScamReport when the user asks about:

Add multiple entities only when the user EXPLICITLY asks for more than one type:
  "properties and scam reports in Miami" → ["Property","ScamReport"]
  "reviews and scams for Tampa landlords" → ["Review","ScamReport"]

------------------------------------
SEMANTIC MAPPINGS — RENT/PRICE
------------------------------------
IMPORTANT: Only set minRent/maxRent when the user provides an EXPLICIT dollar amount.
For vague affordability words, use sortBy instead — do NOT invent a dollar threshold.

Vague affordability words → NO minRent/maxRent, use sortBy:
  "cheap" / "affordable" / "budget" / "inexpensive" / "economical"
  "super cheap" / "very affordable" / "on a tight budget"
  "moderate" / "mid-range" / "reasonable"
  → For Review queries: sortBy: "rent"
  → For Property queries: sortBy: null (no rent sort available); omit rent filter entirely
  → Chip: "💰 Sorted by affordability" (Review) or "💰 Affordability preferred" (Property)

Explicit amounts only → set minRent / maxRent:
  "under $X" / "below $X" / "max $X" / "no more than $X"  → maxRent: X
  "over $X" / "above $X" / "at least $X" / "minimum $X"   → minRent: X
  "around $X" / "about $X" / "roughly $X"                  → minRent: X×0.9, maxRent: X×1.1 (round to 50)
  "expensive" / "upscale" / "premium" / "luxury" / "high-end" / "lavish"
  → These still carry price signal: minRent: 2500 (premium) or 3500 (luxury)

------------------------------------
SEMANTIC MAPPINGS — OVERALL RATING
------------------------------------
"decent" / "okay" / "acceptable" / "not bad"      → minRating: 3.0
"good" / "good ratings" / "well rated"            → minRating: 3.5
"highly rated" / "great" / "very good"            → minRating: 4.0
"excellent" / "exceptional" / "top rated"         → minRating: 4.5
"perfect" / "5 star" / "outstanding"              → minRating: 5.0
explicit "4 stars" / "rated 4 out of 5"           → minRating: 4.0

NOTE: For NEGATIVE qualifiers ("bad ratings", "poor reviews", "worst"), do NOT set minRating.
Instead, keep minRating null and set sortBy or issueTypes to express the intent.

INTERPRETED FILTERS — CHIP FORMAT
------------------------------------
One chip per meaningful extracted constraint. Start each with an emoji:
  📍  city                    → "📍 Tampa"
  🏠  property type           → "🏠 Apartments"

  ⭐  overall rating          → "⭐ Min rating 4.0"
  📞  communication rating    → "📞 Communication min 3.5"
  🔧  maintenance             → "🔧 Maintenance min 4.0" or "🔧 Issues: maintenance, mold"
  ✅  verified                → "✅ Verified stays only" or "✅ Has verified reviews"
  ⚠️  scam type/severity      → "⚠️ Deposit scams" or "⚠️ Severity ≥ 3.5"
  📅  date filter             → "📅 Reported after 2024-11-01"
  🔍  landlord/keyword        → "🔍 Landlord: SunCoast Properties"
  🏷️  issue tags              → "🏷️ Issues: mold, pests"
  📊  sort order              → "📊 Sorted by rating"

Keep chips concise (under 40 chars each). Omit chips for null/false/default fields.

------------------------------------
HARD RULES
------------------------------------
1. Return exactly one JSON object. No other text.

3. Only include entity keys in queriesByEntity that are in targetEntityTypes.
